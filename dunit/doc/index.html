Home of DUnit:
An Xtreme testing framework for Borland Delphi programs

You probably got here through my article in Borland Community Developer News. If not, and you want to know what this is all about, you can read the article here.

DUnit is an Xtreme testing framework for Borland Delphi programs. It was inspired on (actually almost cloned from) the JUnit framework written in Java by Kent Beck and Erich Gamma. You can read all about Xtreme Testing and Extreme Programming from this site.

The current version of DUnit is 3.2.2. See the Log below for the latest changes.

Contributing

If you want to contribute to improving DUnit, please drop me a note just to make sure no amount of work gets duplicated. Right now, the work remaining to do in DUnit is:

  • Translate JUnit's excellent documentation and examples to Delphi.
  • Improve the framework so the user can select the tests to run using the GUI test runner.
  • Provide a way of logging the results of test runs and optionally rerunning only the failed test cases.
  • Package the framework in binary form so it's ready for use.
  • Whatever you come up with...

Contributors

  • Kent Beck and Erich Gamma developed the original testing frameworks in Smalltalk and Java.
  • Juanco Añez <mailto:juanco@suigeneris.org > did the port to Borland Delphi and provided the collections library.
  • Mark Watts <mwatts at taxwise dot com> provided a fix that let DUnit compile under Delphi 5.
  • Tres Seaver <tseaver at palladion dot com> provided a component-based DUnit implementation.
  • Harley Pebley <Harley at medicalpriority dot com> updated the source comments to make them compatible with Time2Help, and provided a Help file created with the tool.
  • Chris Morris <ChrisM at SNELLINGCORP dot COM> has provided several enhancements and bug fixes.
  • Andrew McIntyre <bgc at m140 dot aone dot net dot au> wrote AutoDUnit.pas, a registry for automating the execution of all the test cases in a project.

Download

You should read the license before you download the software. The complete source code is here. The current version is 3.2.2. DUnit works with Delphi v 4 and later.

As it stands, DUnit is vary lacking on documentation. If you can read Java code, then you can get along by reading the documentation that comes with JUnit.

You can also download DUnit_PEM, Tres Seavers's component-based version of DUnit. DUnit_PEM uses a more traditional Delphi / component approach (i.e., use Properties / Events / Methods, rather than Interfaces / Inheritance). This allows easy integration of the test framework into the Delphi IDE -- test suites, cases, and results are simple non-visual components. Some variation along these lines might usefully backport to Java for integration into an EJB environment (which uses essentially the same P/E/M model, borrowed from Delphi). DUnit_PEM is freely distributable open source. Read the enclosed LICENSE.html file for more information.

Log

date version comment
2000/06/29 3.2.2

Moved the msg strings in ITestCase.assertXXX methods to the end, made them default parameters with empty string values, and removed the non-message versions of the methods. These changes cut the interface roughly to half, and provide for a more Delphi-like way of doing things. The changes will break some existing code, but the fixes are very easy.

Added assertEquals and asserNotEquals methods that accept native strings to TTestCase, as by suggestions made by Chris Morris.

Incorporated changes made to the GUI by Chris Morris. Now there are "Select All" and "Deselect All" buttons that can be used to enable/disable all the tests in a suite or sub-suite.

Incorporated the Makefile used to package the distribution zip file to the distribution.

2000/06/20 3.2.1 More bug fixes in the Collections unit.

Added a sketchy Algorithms unit.

Tweaks to the GUI.
2000/06/16 3.2.0 Added the ability to enable/disable tests or test suites from the GUI. The test tree shows different colors for tests that run, suceeded, failed or produced an error.

Added a screenshot of the new GUI to this page.

Fixed several bugs in the collections library, and added test cases to check against the same errors in the future:

  • THashMap was essentially not working.
  • TAbstractListIterator.remove behaved randomly.
2000/06/15   DUnit has an mailing list at eGroups. To join the list, please visit the DUnit Page at eGroups, and select the "subscribe" option. You can also send e-mail to dunit-subscribe@egroups.com to subscribe.
2000/06/13 3.1.0 Added many of the improvements in JUnit 3.1 to DUnit, including:
  • Added an ITestListener interface that clients implement to receive information about the progress of a test run, and reimplemented the text and GUI versions of the test runners to use it.
  • You can use testSuitOf(TTestCaseClass) to automatically generate a test suite out of its  published testXXX methods.
  • The TTestCase.suite class method returns a test suite built using testSuiteOf().
  • TTestSuites can have names.
  • The GUI now displays a tree with the tests being run.

Other changes were:

  • Moved the test case registration routines into DUnit.pas.
  • Added helper routines (like RunRegisteredTests) so tests can be run more easily.

Completely refactored teh collections library used by DUnit to make it more useful in other projects. Enhancements include:

  • ISortedSet and ISortedMap implemented as red/black trees in TTreeSet and TTreeMap.
  • All the interfaces and classes were placed into a single Collections.pas unit.
  • Contained objects don't necessarilly have to implement IObject. The library will behave reasonable with any IUnknown. 
2000/01/06 0.1.7

Andrew McIntyre wrote AutoDUnit.pas, a registry for automating the execution of all the test cases in a project.
1999/12/21 0.1.6

Chris Morris made some enhancements to the GUI:

  • Now status is updated even if there were no failures.
  • The test name is included along with the other information in the list of failures.
1999/12/10 0.1.5

The DFM file was out of Sync with .PAS module. Fix and enhancements provided by Chris Morris and Harley Pebley.

Harley Pebley modified the comments to make them compatible with Time2Help, and provided a Help file created with the product.
1999/12/03 0.1.4 Added missing .DFM file to the distribution.
1999/12/02 0.1.3 Added a GUI interface to DUnit. See the files DUnitGUI and TestDUnitGUI for details. Now calling TTestSuite.Create(classType) will automatically add test cases for all methods in classType with names that start with 'test'.

Tres Seaver created a new implementation of DUnit using components called DUnit_PEM. Download available.

Incorporated bug fix from Chris Morris. Due to the way Delphi implements interfaces, TTestSuite could not be cast to ITest. Trying create suites of suites produced an error. This is now fixed.
1999/11/30 0.1.2 Incorporated bug fixes provided by Mark Watts. Now DUnit compiles under Delphi v5.x
